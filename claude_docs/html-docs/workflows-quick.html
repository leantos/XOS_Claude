WF-01: Opera Income Audit Workflow

Steps

  • Data Import ? OperaIncomeAuditService.AutoFillData()
  • Validation ? Check statistics, revenue, balances
  • Review ? User reviews discrepancies
  • Routing ? WorkFlowMasterService.RouteToNext()
  • Approval ? Manager approves/rejects
  • Finalization ? Update status, send notifications

Decision Points

  • Auto-fill success? ? Continue/Manual entry
  • Discrepancies found? ? Route for review/Auto-approve
  • Manager decision? ? Approve/Reject/Return

Services Involved

  • OperaIncomeAuditService (primary)
  • WorkFlowMasterService (routing)
  • EmailService (notifications)
  • ReportService (audit report)

WF-02: Refund Processing Workflow

Steps

  • Initiation ? RefundService.CreateRefund()
  • Validation ? Check amount, guest details
  • Approval Level 1 ? Supervisor review
  • Approval Level 2 ? Manager review (if > threshold)
  • Processing ? Generate refund transaction
  • Completion ? Update records, notify

Business Rules

  • Amount < 500: Single approval
  • Amount >= 500: Dual approval
  • Credit card: Auto-process after approval
  • Bank transfer: Queue for BACS

Services Involved

  • RefundService (primary)
  • BacsRefundService (BACS processing)
  • WorkFlowMasterService (approvals)
  • EmailService (notifications)

WF-03: BACS Refund Workflow

Steps

  • Collection ? Gather approved refunds
  • Batch Creation ? BacsRefundService.CreateBatch()
  • Validation ? Verify bank details
  • File Generation ? Create BACS file
  • Transmission ? Send to bank
  • Confirmation ? Update status

Batch Rules

  • Daily batch at 14:00
  • Min batch size: 1
  • Max batch size: 500
  • Cut-off time: 13:30

WF-04: Bank Reconciliation Workflow

Steps

  • Import ? BankDataService.ImportStatement()
  • Auto-Match ? BankReconciliationService.AutoMatch()
  • Manual Match ? User matches remaining
  • Exception Handling ? Flag unmatched items
  • Review ? Supervisor review
  • Completion ? Close reconciliation

Matching Rules

  • Exact amount + date: Auto-match
  • Amount within 1%: Suggest match
  • Date within 3 days: Possible match
  • No match: Manual intervention

Services Involved

  • BankReconciliationService (primary)
  • BankDataService (import)
  • ReportService (reconciliation report)

WF-05: F&B Void Workflow

Steps

  • Request ? FBVoidService.CreateVoid()
  • Validation ? Check transaction exists
  • Manager Review ? Verify reason
  • Approval/Rejection ? Decision recorded
  • Processing ? Update POS system
  • Audit Trail ? Log all actions

Void Rules

  • Same day: Auto-approve if < $50
  • Previous day: Requires manager
  • > 2 days: Requires GM approval
  • Multiple voids: Flag for review

WF-06: Night Audit Checklist

Steps

  • Initialization ? NightChecklistService.StartChecklist()
  • Task Execution ? Complete each item
  • Validation ? Verify completeness
  • Sign-off ? Night auditor confirmation
  • Report Generation ? Create audit report
  • Distribution ? Email to management

Checklist Items

  • Close POS day
  • Run credit card batch
  • Generate reports
  • Balance cash
  • Review exceptions
  • Update forecasts
