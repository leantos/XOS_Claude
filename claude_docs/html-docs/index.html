How to Use This Documentation

üìñ Dual-Purpose Documentation System

This documentation works in two ways:

  • Web Browsing: Read online at this site for learning and reference
  • Claude Code Integration: Reference via ../ in your projects

For Development Teams

Add this documentation to your XOS project as a Git submodule:

Add Documentation Submodule 
# In your project root directory
git submodule add https://github.com/your-org/xos-docs.git claude_docs
git commit -m "Add XOS documentation"
git push

Team members clone with: git clone --recurse-submodules [repo-url]

With Claude Code

Reference documentation files directly in your prompts:

Claude Code Reference Examples 
Create user login using ../CRITICAL_PATTERNS.md

Fix input issues with ../frontend/xos-input-handling-fix.md

Build API following ../backend/backend-blueprint.md

Always include ../CRITICAL_PATTERNS.md in your prompts

Keeping Updated

Pull latest documentation when convenient for your team:

Update Documentation 
# Check for updates
cd claude_docs && git fetch && git log HEAD..origin/main --oneline && cd ..

# Pull latest version
git submodule update --remote claude_docs
git add claude_docs && git commit -m "Update XOS documentation"

You control when to update - no automatic changes to your project

Pro Tips for Developers

Do This

  1. Always copy-paste the prompts above - they're tested and work
  2. Be specific about problems - "Username input won't type" vs "it's broken"
  3. Reference exact file paths - ../path/file.md
  4. Let Claude handle the complexity - just point it to the right docs

Don't Do This

  1. Don't assume Claude knows XOS patterns without references
  2. Don't try to explain XOS yourself - let the docs do it
  3. Don't skip the ../CRITICAL_PATTERNS.md reference
